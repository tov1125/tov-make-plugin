## 소프트웨어 개발

### 권장 구성
Planner + Generator + Evaluator (3-agent)

### 평가 기준 예시
| 기준 | 설명 | 검증 방법 |
|------|------|-----------|
| 기능 완성도 | 스펙의 모든 기능이 동작하는가 | Playwright/자동 테스트 |
| 코드 품질 | 가독성, 구조, 에러 처리 | 정적 분석 + 코드 리뷰 agent |
| UI/UX | 사용자 흐름이 직관적인가 | 스크린샷 기반 평가 |
| 통합성 | 컴포넌트 간 연동이 정상인가 | E2E 테스트 |

### 특이사항
- Evaluator에 Playwright MCP를 부여하여 실제 페이지와 상호작용하며 테스트하는 것이 효과적
- API endpoint, DB 상태까지 검증하도록 Evaluator 범위를 확장할 것
- Planner가 AI 기능 통합 기회를 스펙에 포함하면 결과물 품질이 올라감

## 데이터 분석 / ML

### 권장 구성
Generator + Evaluator (2-agent) — 대부분 Planner 불필요

### 평가 기준 예시
| 기준 | 설명 | 검증 방법 |
|------|------|-----------|
| 정확성 | 분석 결과가 데이터와 일치하는가 | 샘플 검증 스크립트 |
| 해석 가능성 | 비전문가도 결론을 이해할 수 있는가 | 수동 리뷰 |
| 재현 가능성 | 동일 데이터로 동일 결과가 나오는가 | 자동 재실행 |
| 시각화 품질 | 차트가 데이터를 정확히 전달하는가 | 스크린샷 기반 평가 |

### 특이사항
- 데이터 파이프라인은 중간 결과를 파일로 저장하여 agent 간 전달
- Evaluator가 통계적 검증(p-value, 신뢰구간)을 수행하도록 하면 환각 방지에 효과적
- Context reset이 적합 — 각 분석 단계가 독립적인 경우가 많음

## 콘텐츠 생성

### 권장 구성
Generator + Evaluator (2-agent)

### 평가 기준 예시
| 기준 | 설명 | 검증 방법 |
|------|------|-----------|
| 톤/스타일 일관성 | 브랜드 가이드에 부합하는가 | 수동 리뷰 |
| 독창성 | 템플릿 냄새가 나지 않는가 | 수동 리뷰 |
| 구조 | 논리적 흐름과 가독성 | 자동(섹션 구조) + 수동 |
| 정확성 | 사실 오류가 없는가 | Fact-check agent 또는 수동 |

### 특이사항
- 평가 기준의 단어 선택이 결과물 방향을 직접 결정함 — "세련된"과 "대담한"은 완전히 다른 결과
- Evaluator가 지나치게 관대해지기 쉬운 도메인 — "구체적 문제를 먼저 나열하고, 점수는 마지막에" 패턴 필수
- 5-15회 반복이 일반적 — Generator가 피드백 기반으로 수정 vs 방향 전환을 전략적으로 선택하도록 할 것

## 연구 / 리서치

### 권장 구성
Planner + Generator + Evaluator (3-agent)

### 평가 기준 예시
| 기준 | 설명 | 검증 방법 |
|------|------|-----------|
| 포괄성 | 주요 관점/소스를 빠뜨리지 않았는가 | 소스 목록 검증 |
| 정확성 | 인용과 사실이 올바른가 | 원문 대조 |
| 분석 깊이 | 표면적 요약이 아닌 통찰이 있는가 | 수동 리뷰 |
| 실행 가능성 | 결론에서 구체적 행동이 도출되는가 | 수동 리뷰 |

### 특이사항
- Planner가 연구 질문을 구체화하고 탐색 범위를 정의하는 것이 중요
- 파일 기반 통신이 특히 중요 — 연구 결과를 구조화된 파일로 축적
- Context reset 전략 권장 — 각 소스/관점 분석을 독립 context에서 수행

## 디자인 / 크리에이티브

### 권장 구성
Generator + Evaluator (2-agent), 높은 반복 횟수

### 평가 기준 예시
| 기준 | 설명 | 검증 방법 |
|------|------|-----------|
| 디자인 품질 | 통일된 색상, 타이포그래피, 레이아웃이 뚜렷한 분위기를 만드는가 | 스크린샷 리뷰 |
| 독창성 | 템플릿 기본값이 아닌 의도적 창작 결정이 보이는가 | 스크린샷 리뷰 |
| 기술 완성도 | 타이포 위계, 간격 일관성, 색상 조화, 대비율 | 자동(contrast ratio) + 수동 |
| 기능성 | 인터페이스 목적 이해, 주요 동작 수행 가능 여부 | Playwright 테스트 |

### 특이사항
- 모델은 기술적 완성도는 자연스럽게 잘 하지만, 미학적 독창성에서 약함 → 디자인 품질과 독창성에 높은 가중치
- Evaluator가 실제 페이지를 렌더링하고 스크린샷을 찍어 평가하는 것이 효과적
- 5-15회 반복 사이클, Generator가 수정과 미학적 방향 전환을 전략적으로 선택
- 기준 문구가 시각적 수렴 방향을 결정 — "museum quality"와 "playful" 은 완전히 다른 결과물
